# Specification: Elite Bootstrap Monte Carlo Engine

## 1. Executive Summary
This document defines the architecture for a probabilistic simulation engine based on the **Portfolio Visualizer (PV)** methodology. It replaces linear backtesting with 10,000 synthetic futures generated by "shuffling" historical market years from the Simba dataset (1972–2025).

## 2. Strategic Objectives
- **Preserve Market Clumpiness:** Use "Full-Year Sampling" to maintain correlations between asset classes (e.g., stocks and REITs crashing together).
- **Capture Regime Risk:** Implement "Block Bootstrapping" to simulate sustained economic periods (inflation, stagflation).
- **Quantify Success:** Calculate probability scores for customizable survival goals (Wealth Preservation, Milestone Achievement, Income Sustenance).
- **Sequence Stressing:** Optionally force the worst-performing years to the beginning of the simulation to test "Sequence of Returns Risk."

## 3. Methodological Core

### 3.1. The Sampler (The "PV" Methodology)
The engine picks a random year $Y$ from the population (1972-2025). It extracts the returns for **all** assets in the portfolio for that specific year.
- **Sampling with Replacement:** The same year can be picked multiple times in one path.
- **Inflation Adjustment:** Historical CPI from the same year $Y$ is applied to maintain "Real Dollar" accuracy.

### 3.2. Block Bootstrap (Optional)
Instead of picking 1 year, the engine picks a starting year $Y$ and pulls a contiguous block of $N$ years (e.g., 3 or 5 years). This captures serial correlation (e.g., the Tech Bubble bleed).

### 3.3. Simulation Scale
- **Iterations:** 10,000 independent 30-year paths.
- **Performance Goal:** Full calculation in **< 800ms**.

## 4. Evaluation Engine (Success Definitions)

The engine accepts a `SurvivalGoal` configuration:

| Goal Type | Success Condition |
| :--- | :--- |
| **Preservation** | Final Real Value $\ge$ Initial Value |
| **Milestone** | Portfolio Hits Target Amount $X$ by Year $N$ |
| **Withdrawal** | Portfolio Value $> 0$ for duration given $X\%$ annual draw |

## 5. UI & Visualization

### 5.1. The Success Funnel
An `AreaChart` (Recharts) plotting the following percentiles over time:
- **90th Percentile (Ceiling):** "Bull Case" luck.
- **50th Percentile (Median):** The most likely outcome.
- **10th Percentile (Floor):** "Black Swan" resilience.

### 5.2. Probabilistic Verdict
The UI displays a large "Success Score" (e.g., **94% Reliability**).

## 6. Component Architecture
- `monteCarloEngine.ts`: The main orchestrator.
- `historicalSampler.ts`: Logic for Single-Year and Block-based year selection.
- `statsCalculator.ts`: Uses **simple-statistics** to derive percentiles from the 10,000 result array.
- `SuccessFunnel.tsx`: High-density Recharts visual.

## 7. Technical Constraints
- **Strict Logic Separation:** No UI code in the engine.
- **Module LOC:** Every logic file must be **< 250 lines**.
- **Memory Efficiency:** Avoid storing 10,000 full NAV arrays; aggregate results into percentiles immediately to save memory.
